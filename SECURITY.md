# Security Policy

## Supported Versions
This repository is actively maintained and secured on the main branch.

## Reporting a Vulnerability
If you discover a security vulnerability related to the PDF-Ready storefront, infrastructure licensing system, or associated assets, please report it responsibly.

### What to include
Please include the following details:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigation

### Response Time
We aim to respond within 48 hours and provide a resolution timeline within 7 days.

## Scope
This policy applies to:
- GitHub Pages storefront (records-Proof/Pdf-ready)
- All PDF-Ready digital product bundles
- Verification and .proof infrastructure assets

## Disclosure Policy
Please do not publicly disclose vulnerabilities until they have been reviewed and resolved.

## Safe Harbor
We will not pursue legal action against security researchers who:
- Act in good faith
- Avoid data destruction or service disruption
- Do not access or modify user data
